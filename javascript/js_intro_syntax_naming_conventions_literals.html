JavaScript Syntax, Naming Conventions, Literals

How is JavaScript spelled?

^

JavaScript is a trademark of the Oracle Corporation. It is spelled in Pascal Case, like this: ’JavaScript’. Only JavaScript classnames should be spelled in Pascal Case

How to name JavaScript variables?

^

Using Camel Case. Variables and function definitions should not be named with ’snake’ case, ’kebab case,’ or reserved words.

What characters are legal in JavaScript?

^

digits(0-9), letters (A-Z) and (a-z), underscore (_), dollar sign ($), and most ISO 8859-1 and unicode letters.

What are identifiers?

^

JavaScript identifiers are names for variables, functions, and classes. They can’t contain spaces, or start with numbers or special characters, other than $ or _

How long can JavaScript identifiers be?

^

A code reviewer should understand from the identifer what the variable, function, or object does. 64 characters is too long.

What are reserved words?

^

Reserved words cannot be used as identifiers. You should read the list of reserved words. In the console, ’window’ and ’document’ are reserved words.

How are semi-colons used in JavaScript?

^

Semi-colons instruct the compiler to send statements. The interpreter attemps to add them, but is unreliable. You should include them.

Where to include semi-colons?

^

Everytime there is a new command or an end of statement, end with semi-colon.

How to declare a function?

^

Function declarations start with the ’function’ reserved word, followed by an identifier, parens() for arguments, and square brackets for body{}

Do conditional statements need semi-colons?

^

’if else’, ’for loops,’ ’while loops,’ and ’switch’ statements do not need semi-colons after them.

How to write comments in JavaScript?

^

Single line comment starts with / multi-line comments start with /* and end with *

How are white spaces and line breaks interpreted?

^

JavaScript is interpreted from top to bottom. White spaces and line breaks don’t matter (use as many as you like)

What is a JavaScript literal?

^

If a variable is assigned a value, the value is called the ’literal’. It’s the ’literal’ expression of the value x=5; 5 is an integer literal.

How many types of literals are there?

^

There are many JavaScript literals. Example:

  • string literals (text wrapped in “)
  • boolean literals (true or false)
  • floating point literals (numbers with decimals)
  • integer literals (negative/positive whole nubmers)
  • array literals (lists in [] with commas)
  • object literals (see later)

What is an array?

^

An array is a comma seperated list enclosed in square brackets. There are special functions and properties available to arrays.

What are object literals?

^

Object literals are lists of pairs of keys and values enclosed in curly braces. Note: key and value are seperated from next pair with comma, and from each other with semi-colon

What are differences between JavaScript objects and JSON

^

One difference is that keys in JavaScript objects are not enclosed in “”, but in JSON data format they are.
