Objects

What types of data are objects good for storing?

^

Because they store data in key value pairs, called ’tuples’, objects are good for storing multi-dimensional uniquely named data types

Do positions matter in objects like they do in arrays?

^

Positions do not matter in objects like they do in arrays. To change, retrieve, or reference tuple values, data tuples in objects are identified by their key values, which must be unique.

What other names and characteristics do values in objects have?

^

Objects can contain private (only referenced from the object) characteristics called ’properties’ and private functions called ’methods’.

How to reference properties and methods of objects?

^

Private properties and methods of objects are referenced by listing the objects name followed by the period ’.’ operator, followed by the identifier of the property or method.

^

Methods have () next to identifier, that can contain arguments

How to add a property of method to an object after it has been declared?

^

list the object and property or method identifier on the left-hand side of an assignment operator, like ’=’, separated by the period operator ’.’ and put the new value on the right-hand side.

^

const anotherObject = {};
anotherObject.name = “This is another object”;
anotherObject.returnName = function (){
alert(anotherObject.name);
};

Show three ways to create an object as a side-effect of another method

What does Object.prototype.create() do?

^

Object.prototype.create() creates a new object from an existing object.
Note it does not copy the object, but links it to the original object through prototypal inheritance.

^

The new object does not create any methods or properties, it just accesses them on the original.

const originalObject = {
key1: “value1”,
key2:function(){
console.log(this.key1);
}
}

What are the three characteristics of object properties?

^

Object properties can be: writeable,
enumerable,
and configurable.

^

You can set the characteristics of object properties using the second argument of Object.create().

^

The first argument takes the name of the copied object. The second argument takes an array of key value pairs, in which the value can be an array that includes key value pairs like:

^

{
writable: true,
enumerable:true,
configurable:true
}

Create an object with a name property. Copy with Object.prototype.create(), make the ’name’ property writeable, configurable, and enumerable

^

let anObject = {
name:“The Coolest Object”
};

^

let aCopyOfObject = Object.prototype.create(
anObject, {name: {
value: “The Coolest Object Copy”,
writeable:true,
enumerable:true,
configurable:true
}}
);

What does making an object property writeable do?

^

A non-writeable object cannot have its properties or methods changed,

(or have properties or methods added to it without ’getters’ and ’setters’).

It also cannot have its configurable value changed.

What does making an object property enumerable do?

^

An enumerable object property can be viewed if it is iterated using the ’for…in’ loop , or Object.keys() method

What does making an object property configurable do?

^

A non-configurable object cannot have its writeable, enumerable, or configurable properties changed

except in the case when it is writeable, then the configurable property can be changed.

Do JavaScript Classes mimic Object.prototype.create()?

^

JavaScript classes use Object.prototype.create() under the hood to link prototype chains to object instances

What are members on the value that is the object?

^

Properties are called ’members’ ’on’ the value that is the object.

What does it mean to say the properties and methods of an object are private?

^

The properties and methods of an object cannot be accessed without referencing the object’s identifier, unless the object is the global object.

^

They are only accessible through reference to the object.

What is Object.prototype?

^

Object.prototype is the standard built-in object with methods and properties on it that can be accessed through prototypal inheritance by any other object in the JavaScript runtime

What does Object.freeze() do?

^

Object.freeze() takes an object as the argument and returns it with the object characteristics, ’writeable’ and ’configurable’ set to false.

^

The purpose of Object.freeze() is to restrict access to objects.

^

You might want to restric access to an object if it contains for example financial data. Locking it in an immutable state would better simulate the permanent characteristics of transactional data.

What does Object.getOwnPropertyNames() do?

^

Object.getOwnPropertyNames()
returns the names of the keys of the properties and methods of a given object.

What does Object.hasOwn() do?

^

Object.hasOwn()
takes an object instance as the first argument and a property as the second argument, and returns ’true’ if the object has the property passed, and false otherwise.

^

[Note: every declared object other than the built-in standard objects is an object instance.]

What does Object.keys() do?

^

Object.keys(objectName);

returns an array of the keys of a given object (the name of which is the first argument).

^

Since arrays are iterable, the array with the object’s keys can now be iterated through.

What itererators are often used with Object.keys(), Object.values(), and Object.entries()?

^

Array.prototype.forEach()
and the ’spread operator’ […]

are often used to iterated through arrays of an objects keys, values, or key/value pairs, for mutations or use in calculations.

What does Object.values() do?

^

Object.values()

takes an object instance as its argument and returns an array of the values.

^

The array can be used with Object.forEach(), the ’spread’ operator, and such loops as the ’for of loop,’ which works on iterables (as opposed to the ’for in loop’, which does not)

What does Object.entries() do?

^

Object.entries()

returns an array of key and value pairs (tuples).

^

The array can be iterated through with Object.forEach(), the ’spread’ operator […], or a ’for of loop’ since its an iterable.

Given an object containing key value pairs, loop through the entries with a ’for of’ loop, and log the values to the console

^

const items = {
livingRoom: ’couch’,
kitchen: ’stove’,
bedroom: ’bed’,
office: ’desk’
};

^

for (let [key,value] of Object.entries(items)){
console.log(`${value}`);
}
