Object and Array Destructuring

What is destructuring?

^

Destructuring is the programmatic process of extracting data from a data structure by matching a pattern on the left-hand side of a statement with a structure on the right-hand side.

What is the advantage of desctructuring?

^

Destructuring creates cleaner code and simplifies operations that can be typically complex, like extracting data from large data structures or modules.

What is array destructuring

^

Array destructuring patterns place variables in positions on the left-hand side of a declaration assignment statement

Destructure the first three positions of an array to the variables a, b, c

^

const numbers = [1,2,3,4,5];
let [a,b,c] = numbers;

^

console.log(a,b,c);//returns 1 2 3

How does array destructuring work?

^

The square brackets containing variable identifiers on the left-hand side of the declaration and assignment statement tell JavaScript to match the left-hand side pattern with the right-hand side pattern and ’destructure’ the data into literals.

How to get to the sixth position of an array using array destructuring?

^

const numbers = [1,2,3,4,5,6,7];
let [,,,,,,a] = numbers;

^

as you can see, every position leading up to the 6th position (containing number 7) must be referenced

How to get to the sixth position of an array using object destructuring?

^

const numbers = [1,2,3,4,5,6,7];
let {6:a} = numbers;

^

When destructuring an array using object destructuring, include the invisible key (index) of the position you want to destructure and use curly braces

What is a ’binding’ destructuring pattern?

^

When a destructuring statement both declares and assigns variables from a data structure, the destructuring pattern is called a ’binding’ pattern.

^

When variables are declared without assignment and later assigned with a destructuring pattern, the destructuring pattern is called an ’assignment’ pattern

Destructure the numbers constant with an assignment pattern

^

const numbers = [1,2,3,4,5,6,7];
let a,b,c;
[a,b,c] = numbers;

What does the spread operator do?

^

When the spread operator prefixes an array identifier, it iterates through the array and returns each value.

^

By wrapping the array identifier and the spread operator with array or object brackets, a new array or object containing the spread elements is returned.

What is difference between spread operator array syntax and spread operator object syntax

^

The curly braces instruct the operator to spread the contents into an object. If the element spread is an array, the keys of the array will be numbers incremented from 0 to the last position.

^

In array destructuring, we use the square brackets and spread the elments into an array.

^

The ’rest’ operator is the ’spread’ operator when it takes the last position among the parameters passed into a function.

Parameters match to arguments by position, the last position can be used to group extra arguments.

^

console.log(letters)}

Set a default argument in a binding destrucing pattern using the `=` operator

^

const letters = [’a’,’b’,’c’];
let [a,b,c,,d=7] = letters;

^

console.log(d);

returns 7, the fourth position doesn’t exist in letters, but there is a default.
Without the default, variable ’d’ would be assigned a value of ’default’

How to pass parameters into a function as an array for destructuring?

^

If you pass an array as the first parameter of a function declaration, the elements of the array of parameters can be used in the function definition.

^

At the callsite, only the first argument will be passed in and destructured (since the first argument is an array)

When passing an array of parameters into a function for a destructuring pattern, can you use fallbacks?

^

Yes, fallbacks or defaults can be inserted with the `=` operator.

function printLetters([a = ’a’, b = ’b’, c]){
console.log(a,b,c);}
printLetters(z,null,undefined);

^

returns z b undefined.

’z’ is passed in, ’b’ is a default, and ’undefined’ is returned in the 2 position, because there is no default set

Can you destructure non-iterable arguments into a function call?

^

No. After passing arrays with values into a function as parameters, they are destructered in the function body, which requires iteration.

^

Non-iterables, like NaN or undefined, will return an error.

What is object destructuring?

^

It is similar to array destructuring. Destructuring patterns on the left-hand side are matched against data structures on the right hand side.

^

However, destructuring patterns are not referenced by position, but by key identifier.

Variables will be declared with identifiers that match key identifiers in the destructured object.

Give an example of object destructuring

^

const obj = {car1:’honda’,car2:’ford’,car3: ’tesla’};

let { car1:red, car2:green, car3:blue} = obj;

console.log(red,green,blue); returns ’honda ford tesla’

What are the ’variable identifiers’ in a destructuring pattern?

^

The ’variable identifiers’ are on the left-hand side of the destructuring statement.

They are the names of the variables that will be assigned or declared and assigned in the statement.

Are parentheses used in object destructuring patterns?

^

Because curly braces {} are overloaded in JavaScript - they are used in functions, blocks, and objects - the destructuring assignment pattern (not the binding pattern) for objects is wrapped in parentheses ’()’.

How can default values be passed into spread operators?

^

A destructuring pattern that expects a value and gets an undefined value can be passed a default value.

^

The default value goes on the right hand side of the equal operator on the right hand side of the || logical or operator.
