Classes

What are JavaScript classes?

^

Classes are special kind of JavaScript object introduced in ES6 that can be used with the ’class’ reserved word to create subordinate objects.

The first letter of a class is always capitalized.

Create an empty class

^

Declaration:
class Manager {};

Expression:
const Manager = class {};

^

Named Expression:
const Manager = class ManagerClass{};

^

With classes, neither declarations or expressions are hoisted, all three ways of declaring classes are the same.

What is prototypal inheritance?

^

Objects that have a subordinate object instance have access to methods and properties on the class object through what is called a ’prototypal chain’

How does an object on the bottom of the prototypal chain inherit access to methods and properties?

^

If an object references a method or property it doesn’t have, it will look for it on the next object upward from it in the prototypal scope chain.

^

If it finds it, it will be called with the ’this’ keyword set to the callsite, if not, it will look upward to the next object in the prototypal scope chain.

^

If not found, the method or property will throw an error.

What does the ’new’ keyword do in JavaScript?

^

  1. The ’new’ keyword creates an empty object
  2. it runs ’Object.create() on the new Object new object referencing the parent class object.
  3. It initializes the new object by running the parent object’s constructor function, which gives the new object the methods and properties specified in the constructor.
  4. It swaps the ’this’ reserved word in the classes constructor function with the newly created object

What does the `extends` reserved word do in JavaScript?

^

The `extends` reserved word is used to create a class that inherits from a parent class.

^

With `extends` you can create a chain of classes that inherit properties from each other.

The top level class might pass some properties to the class below it, and the class below might pass some properties onward to the class below it.

What is the `super` reserved word used for?

^

The `super` reserved word is used to pass properties and methods between constructors.

What are `constructors`?

^

Constructors are functions inside classes containing methods and properties that can be attached to newly created subordinate instances or classes

Give an example of a constructor

^

class Manager{
constructor(){
this.name = ’Jurgen Klopp’;
this.team = ’Liverpool F.C.;
this.ourTeam = function(){console.log(`Our team is ${this.team}`)}
}//end constructor
}//end manager

What happens to the `this` reserve word in a constructor when a subordinate object is instantiated?

^

Because the `this` word always references the call site, when the constructor function is run, the properties and methods in the constructor are bound to the newley instantiated object.

Are methods and properties in constructor functions created with keys and values seperated by colons `:`?

^

Nope. The right-hand side of methods and properties in constructors are separated from the left-hand side with equals assignment operators `=` because they are assignment statements

Are methods and properties on a class that are not inside the constructor added to subordinate instances?

^

The methods and properties in the constructor are added to the newly instantiated object.

^

The methods and classes outside of the constructor are available through prototypal inheritance.

What happens when a new class extends a class without a constructor?

^

JavaScript creates a constructor in the background and adds all the methods and properties that were in the parent class to the extended class so that a new object from the extended class contains the same properties and methods it would contain if it were instantiated from the extended classes parent.

How to change properties outside a constructor on a parent class in the class that extends it?

^

Properties outside the constructor of the parent class on the class that extends it can be changed with direct assignment when the subordinate class is extended

^

class Coach extends Manager{
responsibilities = [’training’, ’practice scheduling’, ’performance assessment’, \\ ’game strategy’, ’morale’, ’press relations’];
}

What happens to the `this` reserved word in a constructor in a class that extends a parent class?

^

JavaScript will throw an error, because any object instantiated from the extended class will not know if it should extract methods and properties from the extended class or the extended classes parent.

How can the `super` reserved word be used in the constructor of an extended class?

^

A constructor in an extended class can take methods and properties from its parent class as parameters.

^

When the identifiers of those methods and properties are passed to super() within the extended classes constructor any object instantiated from the extended class will bind `this` reserved words from the super() declaration to the extended class.

Can properties that exist on the parent class be overwritten on the extended class?

^

Yes, properties that are not referenced on the extended class’s constructor with the `super` reserved word can be directly assigned in the extended classes constructor to overwrite the property assignments on extended classes parent

Can an extended class’ constructor deconstruct parameters from a data object?

^

Yes. Data can be passed into a constructor from a data object. That means json data can be parsed into a data object and passed to a class so that objects can be instantiated from that class based on json data.

Give an example of deconstructing data into an extended class

^

class Player extends Coach{
constructor({name=’NA’, height=’NA’, weight=’NA’, salary=’NA’, age=’NA’, \\ accuracy=’NA’, power=’NA’, endurance=’NA’})
this.name = name;
this.height = height;
this.weight = weight;
this.salary = salary;
this.age = age;
this.accuracy = accuracy;
this.power = power;
this.endurance = endurance;
}//end constructor
}//end extended class
