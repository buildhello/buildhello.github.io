Arrow functions

Are arrow functions dynamically scoped functions?

^

No. Arrow functions are not dynamically scoped, they are lexically scoped. Dynamic scoping in JavaScript happens with the ’this’ keyword

^

Arrow functions do not have their own bindings to the ’this’ keyword or the ’super’ keyword

Are curly braces needed to in arrow function syntax

^

If the function only returns one line, no curly braces are needed to write it.

If there is only one parameter, no function parentheses are necessary.

Are arrow functions typicall used as methods?

^

Arrow functions can be used in methods when lexical scoping is not a problem

^

when a method contians a function that needs to be deliberately scoped to it, the arrow function is an ideal fix.

Should arrow functions be used in or as constructors

^

No. Arrow functions should not be used in constructors, and cannot be used as constructors

What is the arrow function syntax?

^

Arrow functions do not require the ’function’ reserved word.
The parameter parentheses are written, containing any parameters passed into the function.

^

Then an equal operator `=` followed by a greater than operator `>` with no space between but spaces on either side.

^

Next the opening and closing curly braces containing the body of the function.

^

(parameters) => {function body}

Are the opening and closing curly braces required in arrow function syntax?

^

In arrow function syntax curly braces are recommended, but not needed.
Also, if there is only one parameter, parameter parentheses are not necessary

Is the ’return’ reserved word needed in arrow functions?

^

If there is nothing else in the function body but a one-line ’return’ statement, netither the ’return’ reserved word or the curly braces are needed.

How does an arrow functions solve a lost ’this’ reference in a setTimeout() call in a method on an object

^

When the setTimeout execution context disappears, ’this’ is not bound to undefined, it searches the outer scope of setTimeout’s callback function, which is the scope of the method.

^

The method is a private property of the object that called it, which becomes the target for the ’this’ reference.

If you use an arrow function to declare a method (rather than a callback function within a method), can you use ’enhanced object literal’ syntax?

No, you must use a key identifier with the method.

^

It is probably faster to use the ’enhanced object literal’ syntax, since arrow functions are meant to be used with lexical functions are needed.
