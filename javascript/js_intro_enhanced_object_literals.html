Enhanced object literals

What are ’enhanced object literals’?

^

’Enhanced object literals’ provide a shorthand syntax for declaring properties and methods in objects.

^

They also allow for ’computing property names with expressions’, making calls to the ’super’ operator, ’setting’ the objects ’prototype’ and use of ’Symbol.iterator’

Describe the ’shorthand’ properties of ’enhanced object literal’ syntax

^

The shorthand property syntax of enhanced object literals allows for removal of value specification when the key and the value of a key/value tuple are the same.

Give an example of enhanced object literal syntax

^

When a key/value pair in an object repeats identifiers (the value is a variable and the key has the same name), enhanced object literal syntax can be used to shorten it. The colon can be dropped, and the key is not necessary.

^

let height = ’5 ft 7 in’;
let age = 36;
const player = {
height,
age
}

^

Original syntax:
const player = {
height: height,
age: age
}

What is the enhanced object literal syntax for methods?

^

The ’function’ reserved word is not required. All that is necessary is an identifier prefixing the method’s parameter parentheses.

^

let height = ’5 ft 7 in’;
let name = ’Jim Koders’;

^

const player = {
height,
name,
getName(){
console.log(this.name);
}
}

What are enhanced object literal ’computed property names’?

^

Computed property name enable the programmer to compute the property names.

^

The key side of the key/value tuple in an object’s property is enclosed in square brackets `[]`.

The brackets can contain a valid variable identifier or an expression that returns a value or string.

Summarize enhanced object literal syntax

^

The syntax improvements make it possible to shorten properties declarations when keys and values have the same name, and to shorten method declarations as well.

^

Computed property names make it possible to create objects more programatically than before.

^

Enhanced object literals can be used to set an object’s prototype, make calls using the ’super’ operator, and set a ’[Symbol.iterator]’
