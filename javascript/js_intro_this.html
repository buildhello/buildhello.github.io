What does lexically scoped mean?

^

When scope is set when a function is defined (declared) rather than when it is called, the function is said to be lexically scoped.

^

Functions in JavaScript are lexically scoped by default.

What does dynamically scoped mean?

^

When scope is set when a function is called rather than when it is defined, the function is said to be dynamically scoped.

^

Dynamic scoping in JavaScript is achieved with the ’this’ reserved word.

Do objects have scope in JavaScript?

^

Objects in JavaScript do not have scope. There are only three scopes, function scope, global scope, and block scope (introduced in ES6)

What does the ’this’ reserved word do when used in a function in an object?

^

The ’this’ reserved word points to the object instance or function instance that called it.

^

It can then be used to reference other properties or methods on the object.

Does ’this’ reference the object that defines it or the object that calls it?

^

’this’ looks like it references the object that defines it, but it actually references the object that called it.

^

’this’ always points at an object, either the object that called it, or the function object that called it. This does not reference the scope of an object, it points to an object.

What does the ’this’ reserve word introduce to JavaScript?

^

’this’ introduces dynamic scoping to JavaScript. Dynamic scoping is set when the object or function object is called, rather than when it is defined.

What is a ’with’ statement in JavaScript, and how to use it to reference properties of a property of an object

^

with(root.A) {

^

Note: ’with’ is prohibited in strict mode and comes with some performance impact. Use ’this’ or destructure.

What does ’this’ references when called from a function nested within a method?

^

When ’this’ is called from a function declared within a method, it references the function scope it is in, not the Object that called the method.

In such a case it is frequently lost.

What happens in non-strict mode if ’this’ is ever set to ’undefined’ or ’null’

^

In non-strict mode, if ’this’ is ever set to ’null’ or ’undefined’ it defaults to globalThis, which in the browser is the window object.

What does ’losing this’ mean?

^

When this cannot find a reference, which happens with functions window.setTimeout(), Array.prototype.find() or Array.prototype.forEach(), and promises, it is set ’undefined’ or ’null’, and then defaults to the globalThis in non-strict mode

Give an example of ’losing this’ using window.setTimeout()?

^

Since the execution scope of the callback function disappears at runtime, its ’this’ value is set to undefined, and then bound to the ’globalThis’ object.

^

  • The ’this.name’ reference returns ’undefined.’ ’this.name’ is actually pointing at the anonymous function that called it, but that function no longer exists, only its closure exists.

What happens after 1 second when setTimeout() is called from within a function on an object?

^

  • delay on setTimeout() expires
  • closure containing setTimeout() callback function executes
  • there is no longer a default ’this’ reference
  • ’this’ is set to ’undefined’, then ’globalThis’
  • since the properties or methods of the object ’this’ was meant to point at don’t exists. ’undefined’ is returned

Is lost ’this’ set to point at the globalThis object in strict-mode?

^

No. Lost ’this’ is not set to point at the globalThis object in strict mode, it returns an error

List three methods on the built-in Function object that set a pointer for ’this’

^

  • Function.prototype.call()
  • Function.prototype.apply()
  • Function.prototype,bind()

Are Function.prototype.bind(), Function.prototype.call(), and Function.prototype.apply() all executed at runtime?

^

Yes. Only Function.prototype.bind() adds the ’this’ reference to the closure where it is returned in situations when it would normally be lost.

What are the differences between Function.prototype.call() and Function.prototype.apply()?

^

Function.prototype.call() and Function.prototype.apply() are both applied at the callsite and invoke a function with a ’this’ context passed in as the first argument and any parameters the function has passed to it in the second argument.

^

  • Function.prototype.apply() takes the parameters in an array
  • Function.prototype.call() expects individual arguments delimited with commas

How are arrow functions relevant to the ’this’ reserved word?

^

Arrow functions do not recognize the ’this’ keyword.
Arrow functions with ’this’ references are not dynamically scoped they are lexically scoped
