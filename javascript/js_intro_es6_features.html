What is a template literal string?

^

Template literals are used to construct strings composed of string sections, variables coerced to strings, expressions coarced to strings, and/or functions composed as strings.

^

The ’`’ opens the template. Any ${} containing expressions, variables, or functions are coarced to strings, and the ’`’ closes the template

Can template literals produce formats other than strings?

^

Yes, tagged template literals can be used to produce formats other than strings.

Can template literals be used for strings componsed of multiple lines and spaces?

^

Yes, template literals can be used for strings composed of multiple lines and spaces.

Can you use complex expressions with loops and iteration methods in the template literal interpolation context?

^

Yes. With an IIFE expression, loops and iteration methods can be used in the template literal interpolation context.

^

${(function (){
loop
})()
};

^

however, it’s generally not recommended to do so.

How to use defaults when declaring properties of objects?

^

Both Logical or `||` and the equal operator `=` can be used to set defaults in object declarations.

But `=` only works if the lef-hand side is `undefined` or `NaN`

^

const ddate = {
name: undefined || ’Huey’,
weight: null = ’150lbs’
}

Is the default value needed in object declaration or function declaration?

^

The default value is needed to handle situations when data sources are incomplete or containing ’falsy’, ’undefined,’ or ’null’ values.

^

Defaults are needed in functions that operate on objects.

Use the || default in an object’s method

^

const player = {
name: undefined,
listName: function (){
console.log(`My name is ${(this.name || ’Ronaldo’)}`)
}
}

How to pass default parameter values into a function definition?

^

When you declare comma separated parameters, use the `=` operator after each one that will take a default value.

^

function test(score = ’80’, time = ’200’, room = ’gymnasium’){ console.log(` the score is ${score}. The time is ${time}. The room is ${room}`);}

What is the difference between default parameters in object declarations and in function declarations?

^

In the object declaration the `=` on provides a default if the lef-hand side is ’NaN’ or ’undefined’.

^

In the parameter default, the `=` operator provides a default only when the left-hand side is ’null’ or ’undefined’, ’NaN’ is parsed as a value.
