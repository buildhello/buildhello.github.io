Modules

What are the advantages of using modules?

^

Modules allow units of code to be reused on specific pages. They are useful for sharing code across departments, organizations, and people.

^

They also make it possible to control public/private code access, which is import for keeping the global namespace clean

What is the relationship between the IIFE pattern and ES6 modules?

^

IIFEs, or ’immediately invoked functions’, behave like modules in that they create a namespace for code within a function. In fact, ES6 modules compile into IIFE statements

Name some other module systems?

^

Common JS is the most commonly used, since it’s the default module system in the NODE JS framework

When using Node modules can your code end in .js?

^

In Node JavaScript files need to end in .mjs which lets Node know you are using the ES6 module system.

^

You can also use typescript and end files in .ts, which automatically adds ES6 module compatibility.

What are the two ways of writing the IIFE pattern?

^

`(function({}))()`
`(function({})())`

^

They both close over a function creating a function space that is executed immediately

How can export functions from an IIFE?

^

You can create a global object, outside of the IIFE closure, and attach the functions, methods, and arrays you want to export to it.

^

They will then be accessible in the object at the global level

Do you need to load every JavaScript file to the page when you do an import from a module?

^

No. When you do an import from a JavaScript global level module, you only need to load one file into your HTML script.

^

You import the other files, or modules into that file.

Do modules need to be loaded in order?

^

No. Modules do not need to be loaded in order. Live bindings are parsed at compilation time, not runtime.

Do modules need to be deferred?

^

No. Modules do not need to be deferred. As long as type=“module” is specified on the file, modules are automatically deferred.

Are modules in strict mode?

^

Yes, modules are in strict mode automatically.

What does `import` do in a JavaScript module?

^

`import` loads `bindings,` which are declared, assigned or updated functions, classes, objects, data structures, instances, and so on.

^

Once ’bindings’ are imported, they cannot be updated, but they can be used in the importing runtime, wherein they are called ’live’ bindings

What is `named import module syntax`?

^

bindings exported from a module can be imported to another module by name. The named bindings are destructured from the module exports

^

import {button1, clear, aPublicObject} from ’./modules/module1.js’

^

export {button1, clear, aPublicObject};

What is export list syntax in ES6 modules

^

Export list syntax lists export items in a object and exports at the foot of a module

^

export {button1, clear, aPublicObject};

How do default exports work?

^

Any module can have one default export, which can be viewed as having its own ’invisible’ object.

^

It doesn’t matter what name it takes on the import side, but it cannot take the same name as a variable from a destructured module object.
\\\ It can be used alone, or in addition to the named module export pattern.

What is the point of a module?

^

Minimize name collisions
Keep the global namespace clear
Enable better organization of code

Is a default export a named export?

^

The default export can be named:

export {name1 as default, object1, function1};

It can be another piece of code, but there can only be one default

^

`export default expression;`
`export default function functionName`
`export default class className{}`
`export default Objectname`;

Can you export a live event listener from a module?

^

I think you cannot. I have read that you can, but I tried it and it didn’t work that well.

^

You should export the function or element, then and the event listener on it in your global module

Give some examples of import syntax

^

import defaultExport from “module name”;
import * as name from “module-name”;
import {export1} from “module-name”;
import {export` as alias1 } from “module-name;
import {default as alias} from ”module-name“;

^

import {export1, export2} from “module-name”;
import {export1, export1 as alias2} from “module-name”;
import “string name” as alias } from “module-name”;

^

import defaultExport, {export1} from “module-name”;
import defaultExport * as name from “module-name”;
import “module-name”;

Can you rename objects when you import them?

^

Yes. The `as` keyword enables renaming, if you do not like the name of an export in a module.

^

`import {button1, aPublicObject as button2} from ’./modules/module.js’

How can you import everything that was exported from a module?

^

The `*` symbol stands for ’all’, you can use it in a `namespace` import by importing everything that was exported from a module

^

`import * from ’./utilities.js’;

Are module exports evaluated more than once if they are imported more than once in the same module?

^

No. Module exports are only evaluated one time when they are imported. If you want to reuse code, import a function and call it more than once.

Can you import then configure an object into a module?

^

Yes. The one-time evaluation feature can be used to import an object from a module, then change the properties (configure it).

Are module imports evaluated when they are imported?

^

Yes. Module imports are evaluated only once when they are imported. You can use this feature to update the properties and methods of a module.

Can import module scripts be used to import code that needs to be run as soon as it is imported?

^

Yes. Async inline module scripts can be used to import any functionality that needs to run as soon as it is imported.

^

`<script async type=“module”>
import {aboveTheFold} from ’./dropdown.js’;
aboveTheFold.renderMenu();
<script>`

How can you write code for browsers that don’t support modules?

^

The ’nomodule’ attribute can be added to module script tags to provide backups for browsers that don’t support <script> tags with type=“module” (though that should not be an issue any longer)

What are dynamic module imports?

^

Dynamic module imports can be import anywhere in a script.
