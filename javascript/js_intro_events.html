Events

What is a JavaScript event?

^

A JavaScript event occurs when an action is recognized by the JavaScript engine and marked as an event. Events are interesting changes to the operating system and browser that the browser knows about, and can pass information about to JavaScript.

^

In the browser engine, examples include: button clicks, page scrolls, mouse movements, and even key presses.

What can events be used for?

^

JavaScript can use events to change data and make changes to websites

What is an event listener?

^

An event listener can be attached to an event. It waits for the event to occur, then runs some code, usually a function.

Add an event listener to a scroll event that runs the window object’s alert method.

^

window.addEventListener(’scroll’, function(){alert(“Hello”);});

Is setTimeout(); asynchronous or synchronous

^

setTimeout() is a non-asynchronous method on the DOM API. The interpreter stops reading code at each one of the calls to setTimeout() until the call finishes.

What is the DOMContentLoaded event?

^

The DOMContentLoaded event fires after all HTML has been parsed into DOM nodes, ensuring nodes exist when code is run. Remember scripts are more often marked with defer or type=’module’ to ensure nodes exist.

^

In that case, code runs just before the DOMContentLoaded event.

What is the ’readystatechange’ event?

^

The ’readystatechange’ event occurs whenever the readystate attribute of the document changes. It can be used to detect errors or issues with loading resources, such as scripts, stylesheets, images, etc.

^

It’s also used to determine when a document has completely loaded and can be interacted with, similar to load and DOMContentLoaded.

What is the order of readystatechange, DOMContentLoaded, and load?

^

  1. deferred code
  2. readystate interactive
  3. DOMContentLoaded
  4. readystate complete
  5. load

What is the ’action’ attribute on the form element?

^

The ’action’ element can define URL where the form should be sent (like a script file), it can be used also to redirect the user to a page after form submission.

^

If left empty, data will be sent to the same URL as the current page.

What is the method attribute on the form element?

^

The ’method’ attribute specifies the HTTP method use to submit form data.
Most common methods are ’GET’ and ’POST’

^

’GET’ sends form data as part of URL
’POST’ sends form data in the request body. (use POST for sensitive or all data)

What is the ’target’ attribute on a form element?

^

The ’target’ element can be used to open the response in a new window or specific frame.

^

  • _blank response will be opened in a new window or tab
  • if set to specific iframe, will be opened in that frame
  • if not specified will open in same frame or window as form

What target attribute value requires an extra protective attribute, and what is it?

^

target=blank should not be used without a ’rel’ attribute set to ’noopener’ or ’norefere’ rel=’noopener’

What is the ’enctype’ attribute used for?

^

The ’enctype’ attribute is used to specify the format data should be sent in when it is sent to a server. You won’t need it in forms often, unless your form includes one or more file uploads

What is ’application/x-www-form-urlencoded’?

^

the default value of the enctype attribute. Data is sent to the server in the same format as the query string. Preferable choice in majority of situations

What is ’multipart/form-data’?

^

This is the value used when the form includes file upload fields, and the data is sent in multipart format. Form data and files sent in same request. without this setting file uploads will not work.
